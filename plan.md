# Daily Data Update Diagnosis Plan

## 목표

GitHub Pages 사이트는 열리지만 매일 포트폴리오 데이터가 갱신되지 않는 원인을 찾고, 가능한 범위에서 자동 갱신 경로를 복구한다.

## 가정

- 민감한 계좌 데이터는 기존 설계대로 로컬에서만 수집하고, GitHub에는 암호화된 `public/portfolio.enc.json`만 올린다.
- GitHub Actions는 정적 Pages 빌드/배포만 담당하고, 증권사 API 수집은 로컬 PC의 스케줄 작업이 담당한다.
- 실제 계좌 데이터나 비밀 값은 출력하지 않는다.

## 성공 기준

1. 현재 자동 갱신이 멈춘 지점을 확인한다.
2. 로컬 수집, 암호화 payload 생성, git push, Pages 배포 중 끊긴 단계를 구분한다.
3. 수정이 필요하면 최소 범위로 수정한다.
4. `npm test`와 필요한 빌드/명령 검증을 실행한다.
5. 남은 수동 확인이 있으면 정확한 명령과 위치를 기록한다.

## 2026-06-18 화면 자동 새로고침

### 목표

대시보드 화면에서 최신 암호화 payload를 10분마다 자동으로 확인하고, 사용자가 버튼으로도 즉시 확인할 수 있게 한다.

### 가정

- 화면 새로고침은 GitHub Pages에 올라온 `portfolio.enc.json`을 다시 받는 기능이다.
- 정적 Pages 화면에서 증권사 API 수집이나 로컬 PC 작업 실행은 하지 않는다.
- 잠금 해제 후에는 비밀번호를 브라우저 메모리에만 보관하고, 잠금 시 삭제한다.

### 성공 기준

1. 잠금 해제 상태에서 10분마다 encrypted payload를 다시 fetch하고 복호화한다.
2. 수동 버튼으로 즉시 같은 재조회가 가능하다.
3. 마지막 확인 시각과 자동 확인 주기를 화면에서 알 수 있다.
4. payload가 아직 갱신되지 않은 경우 기존 데이터 표시를 유지한다.

## 2026-06-18 홈 폴더 갱신 명령

### 목표

프로젝트 폴더가 아닌 `C:\Users\asher` 같은 위치에서도 npm 명령을 외우지 않고 데이터 수집, 암호화, commit, push를 한 번에 실행한다.

### 가정

- 실제 수집과 배포는 기존 `npm run daily-update` 흐름을 그대로 사용한다.
- 비밀번호는 파일이나 PowerShell 프로필에 저장하지 않는다.
- PowerShell 프로필에는 wrapper 함수를 등록해 명령 진입점만 제공한다.

### 성공 기준

1. `Update-StockMonitoring` 명령이 어느 위치에서나 프로젝트로 이동해 daily update를 실행한다.
2. `PORTFOLIO_PASSWORD`가 없으면 실행 중에만 비밀번호를 입력받고 종료 후 제거한다.
3. push 후 가능하면 GitHub Pages 배포 상태까지 확인한다.
4. 사용 방법이 README와 작업 로그에 남는다.

## 2026-06-18 계좌 요약 제거와 프로젝트 마무리

### 목표

계좌별 요약 화면을 제거하고 포트폴리오 비중 legend를 그 자리로 옮긴 뒤, 남은 계획 항목을 실제 완료 항목과 외부 입력 필요 항목으로 정리한다.

### 가정

- 계좌 데이터는 총 잔고와 현금 계산에 계속 필요하므로 데이터 모델에서는 제거하지 않는다.
- 토스증권 API와 미래에셋 실제 샘플 보강은 외부 접근 권한과 실제 샘플 없이는 완료로 표시하지 않는다.

### 성공 기준

1. 화면에 `계좌별 요약` 섹션이 더 이상 보이지 않는다.
2. 포트폴리오 비중 legend가 기존 계좌별 요약 위치의 독립 패널로 보인다.
3. README, 계획, checklist가 현재 구현 상태와 외부 입력 필요 항목을 정확히 구분한다.
4. 테스트, 빌드, 브라우저 확인, commit, push를 완료한다.
