# Plan

## 목표

한국투자증권, 토스증권, 미래에셋에 흩어진 주식 계좌를 통합해서 수익률과 비중을 쉽게 확인하는 GitHub Pages 대시보드를 만든다.

MVP는 계좌 통합, 평가손익률, 총손익률, 종목별 비중, 목표 비중 대비 리밸런싱 필요 여부에 집중한다.

## 비목표

- GitHub Pages 안에서 증권사 로그인을 수행하지 않는다.
- GitHub에 평문 금융 데이터를 올리지 않는다.
- MVP에서 뉴스, 공시, 리포트, 투자 지식 자동 분석을 만들지 않는다.
- 자동 git commit/push는 로컬 명령 안에서만 실행한다.

## 성공 기준

- 로컬 CLI가 증권사 데이터와 목표 비중을 공통 스키마로 정규화한다.
- 로컬 CLI가 정규화된 데이터를 비밀번호 기반으로 암호화해 `portfolio.enc.json`을 만든다.
- GitHub Pages 앱은 비밀번호 입력 전에는 투자 데이터를 표시하지 않는다.
- 올바른 비밀번호를 입력하면 전체 KPI, 포트폴리오 비중 범례, 종목별 통합 테이블, 리밸런싱 테이블이 보인다.
- 평문 원본 파일, API 키, 로컬 설정 파일은 git에 커밋되지 않는다.

## 구현 순서

1. 프로젝트 스캐폴딩을 만든다.
2. 데이터 스키마와 타입을 코드로 고정한다.
3. 암호화와 복호화 경로를 먼저 구현한다.
4. 암호화된 더미 데이터로 대시보드를 구현한다.
5. 미래에셋 CSV/XLSX 샘플 파서를 구현한다.
6. 한국투자증권 API 어댑터를 구현한다.
7. 토스증권 API 어댑터는 공식 API 사용 가능 상태와 응답 샘플이 확보되면 구현한다.
8. 리밸런싱 계산과 경고를 검증한다.
9. Windows 작업 스케줄러 명령을 추가한다.
10. 빌드, 테스트, GitHub Pages 배포 설정을 마무리한다.

## 현재 마무리 상태

- 한국투자증권 API 잔고와 실현손익 수집은 구현했다.
- 미래에셋은 표준 CSV/XLSX 파일 기반 수집을 구현했다.
- 화면 자동 확인, 수동 새 데이터 확인, 매수금액 표시, 포트폴리오 비중 범례를 구현했다.
- `npm run daily-update`와 `Update-StockMonitoring`으로 로컬 수집, 암호화, commit, push를 한 번에 실행한다.

## 외부 입력 필요

- 토스증권 API 어댑터는 공식 API 접근 권한, 최신 문서, 실제 응답 샘플이 필요하다.
- 미래에셋 실제 샘플 파일 기준 헤더 매핑 보강은 사용자가 내려받은 실제 CSV/XLSX 샘플이 필요하다.
